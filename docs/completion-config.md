# Completion Config

Completion is defined in `lua/plugins/completion.lua`.

## Installed plugins

- `saghen/blink.cmp`
- `saghen/blink.lib`
- `rafamadriz/friendly-snippets`
- `mini.snippets` from `mini.nvim`

## Snippet loading

`mini.snippets` loads snippets from runtime paths using:

- `all.json`, `all.lua`, `all.code-snippets`
- `global.json`, `global.lua`, `global.code-snippets`
- language-specific patterns generated by `by_lang()`

Language-specific search patterns include:

- `<lang>/**/*.json`
- `<lang>/**/*.lua`
- `<lang>/**/*.code-snippets`
- `**/<lang>.json`
- `**/<lang>.lua`
- `**/<lang>.code-snippets`

## Mini snippets setup

- expand/jump mappings are left empty
- Blink owns the completion interaction

## Blink build behavior

Two build paths exist:

1. a `PackChanged` autocmd builds `blink.cmp` with
   `cargo build --release` after install or update
2. `cmp.build():wait(60000)` runs during setup

This means the repo expects a working Rust toolchain for the best Blink path.

## Blink setup

### Keymaps

- preset: `enter`
- `<C-y>` -> `select_and_accept`
- `<C-j>` -> `select_next`
- `<C-k>` -> `select_prev`

### Appearance

- `nerd_font_variant = 'mono'`

### Completion UI

- documentation auto-shows after `200ms`
- menu auto-shows
- menu columns show icon, label, description, and kind
- Tree-sitter is used for LSP menu drawing

### Cmdline

- ghost text is enabled in cmdline completion

### Sources

Default source order:

- `lsp`
- `path`
- `snippets`
- `buffer`

### Extras

- signature help enabled
- fuzzy implementation set to `prefer_rust_with_warning`
